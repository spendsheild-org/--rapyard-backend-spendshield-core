# Contribution Guidelines

Thank you for your interest in contributing to the SpendShield Core repository! We appreciate your efforts and look forward to your contributions. Here are the guidelines to help you get started:

## How to Contribute
1. **Fork the Repository:** Start by forking the repository to your own GitHub account.
2. **Create a Branch:** Create a new branch for your feature or bug fix.
3. **Make Changes:** Implement your changes, ensuring that you adhere to our coding standards.
4. **Commit Changes:** Make small, focused commits with clear messages.
5. **Open a Pull Request (PR):** Open a PR from your branch to the `main` branch of the original repository.

## Code of Conduct
We expect all contributors to abide by our Code of Conduct. This includes:
- Treating everyone with respect
- Being inclusive and welcoming
- Listening to others' opinions and feedback
- Reporting any unacceptable behavior to the project maintainers

## Pull Request Standards
When submitting a pull request, please ensure:
- The PR title is clear and descriptive.
- The PR description includes the motivation and reasoning behind the changes.
- Related issues are referenced in the PR description (e.g., Fixes #issue_number).

## Commit Conventions
To maintain a clean project history, please follow these conventions for commit messages:
- Use the present tense (e.g., "Add feature" instead of "Added feature").
- Use the imperative mood (e.g., "Fix bug" instead of "Fixes bug").
- Include a short summary of the change in the subject line and further detail in the body if necessary.

## Testing Requirements
All contributions must include appropriate tests. Please ensure that:
- New features or changes are accompanied by unit tests.
- Tests are written following the existing testing structure in the repository.
- You run all tests and ensure they pass before submitting your PR.

## Thank You!
Contributions make our project better; thank you for taking the time to help us improve!
